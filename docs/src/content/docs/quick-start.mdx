Get your blog up and running in just a few minutes.

## Run Locally

### Prerequisites

- [Node.js](https://nodejs.org/) v20 or higher
- [Git](https://git-scm.com/)
- A folder with markdown files (e.g., Obsidian vault)

<Steps>

1. **Create your repository**

   Click the **Use this template** button on the [girok-md repository](https://github.com/7loro/girok-md) to create your own repository.

   :::tip[Naming your repository]
   Name your repository `username.github.io` (e.g., `7loro.github.io`) for easy GitHub Pages deployment.
   :::

2. **Clone and install dependencies**

   <Tabs>
     <TabItem label="npm">
       ```bash
       git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
       cd YOUR_REPO
       npm install
       ```
     </TabItem>
     <TabItem label="pnpm">
       ```bash
       git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
       cd YOUR_REPO
       pnpm install
       ```
     </TabItem>
   </Tabs>

3. **Configure your blog**

   Edit the `setting.toml` file:

   ```toml title="setting.toml"
   # Absolute path to your markdown files
   source_root_path = "/path/to/your/obsidian/vault"

   # Blog name
   blog_name = "My Blog"

   # Site URL (for SEO)
   site_url = "https://your-username.github.io"

   # Locale (en or ko)
   locale = "en"
   ```

4. **Sync your posts**

   ```bash
   npm run sync
   ```

   This command scans your markdown folder and copies files with `publish: true` in their frontmatter.

5. **Start the development server**

   ```bash
   npm run dev
   ```

   Visit [http://localhost:4321](http://localhost:4321) to see your blog!

</Steps>

## Quick Start with Docker

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/)
- [Docker Compose](https://docs.docker.com/compose/install/)
- A folder with markdown files (e.g., Obsidian vault)

Prefer Docker? You can run girok-md with **Docker Compose** in 4 steps.

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/7loro/girok-md.git
   cd girok-md
   ```

2. **Prepare a test markdown file**

   Create a markdown file with `publish: true` frontmatter in any folder:

   ```bash
   mkdir -p ~/my-posts
   cat > ~/my-posts/hello.md << 'EOF'
   ---
   title: Hello World
   publish: true
   ---

   My first blog post!
   EOF
   ```

3. **Update the volume path in `docker-compose.yml`**

   Replace `/path/to/your/markdown-folder` with your actual markdown folder path:

   ```yaml title="docker-compose.yml" {4}
   volumes:
     # Example: ~/my-posts, /Users/me/ObsidianVault, etc.
     - ~/my-posts:/source:ro
     - ./setting.toml:/app/setting.toml:ro
   ```

   :::caution
   This step is required. Without a valid path, the container will fail to start.
   :::

4. **Start with Docker Compose**

   ```bash
   docker compose up -d
   ```

   Visit [http://localhost:8080](http://localhost:8080) to see your blog!

</Steps>

:::note
For advanced Docker options (custom builds, volume mounts, rebuild commands), see the [Docker Deployment Guide](/girok-md/guides/deployment#docker).
:::
